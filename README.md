## 🧠 一点心得

第一周最大的收获不是代码写得多漂亮，而是**走通了完整的开发流程**：

在 VS Code 写代码 → 在终端跑通 → git add → git commit → git push → 在 GitHub 看到自己的代码 → 开 GitHub Pages 让全世界访问

也踩了一些经典的坑：

- 第一次 git commit 报错 "unable to auto-detect email address" —— 原来 Git 第一次用要先配置身份
- 文件名带空格的脚本，跑命令时要用双引号包起来：python3 "Temperature converter.py"
- GitHub Pages 默认不开，要去 Settings → Pages 手动选 main 分支才能生效

**核心理解：** GitHub 不是写代码的地方，是放代码的地方。
代码在本地（VS Code）写，写完用 git 推送上去，让别人能看到。
